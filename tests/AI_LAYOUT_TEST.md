# SiYuan AI 排版测试手册

这是一份**给 AI 执行**的排版测试文档。

请调用 `siyuan-mcp-sisyphus` 完成排版。

目标不是考察 AI 会不会写普通 Markdown，而是测试 AI 能否根据 MCP 提供的帮助说明，正确判断思源里的不同排版元素、结构语义与富内容类型，并把它们放到合适的形式里。

---

## 1. 执行目标

请新建一篇测试文档，并把这篇文档排版成一份“思源排版能力展示稿”。

要求：

- 最终结果要像一篇结构完整、可阅读的展示文档。
- 只输出最终排版内容，不要解释你是怎么实现的。
- 如果你不确定某种元素该如何在思源里表达，先读取 MCP 提供的相关帮助资源，再决定实现方式。
- 不要把“能显示出来”当成“语义正确”。本测试更关注元素类型是否选对。

---

## 2. AI 执行规则

1. 优先使用思源原生、语义正确的排版元素。
2. 不要在文档正文里说明具体语法、内部块类型、属性名、底层实现细节或 MCP 推理过程。
3. 不要把以下概念混为一谈：
   - 普通表格 与 结构化数据区
   - 标签 与 收藏语义
   - 一般排版布局 与 并排/卡片式布局
   - 普通代码示例 与 可渲染图形内容
   - 排版结构 与 闪卡语义
4. 每一类元素都要真正出现在文档里，不要只写一句“此处应有某元素”。
5. 文档整体应有统一主题。建议主题为“团队知识库展示页”，但可在不影响测试目标的前提下微调内容题材。

---

## 3. 需要完成的排版任务

### 3.1 基础结构区

请至少包含以下内容：

- 一个总标题与简短导语
- 多级标题结构
- 至少 2 段普通段落
- 一组无序列表
- 一组有序列表
- 一组任务清单
- 一个引述内容区
- 一组不同重要程度的信息提示区
- 一段公式展示区
- 一段代码展示区
- 一条分隔元素

### 3.2 行级表达区

请做一个“文本能力示例”区域，至少包含：

- 超链接
- 图片
- 加粗文本
- 倾斜文本
- 下划线文本
- 删除效果文本
- 高亮标记文本
- 上标与下标
- 行内代码
- 小型行内备注
- 标签示例
- 至少一组层级标签示例
- 至少两种不同的颜色或强调效果
- 至少 3 个表情字符

### 3.3 复合布局区

请做一个“信息对照展示”区域，至少包含：

- 一个左右并排的双栏区域
- 一个包含 4 个内容单元的卡片式区域
- 一个适合放摘要或对比信息的嵌套布局区域
- 一个会展示动态查询结果的区域

注意：这里测试的是布局是否正确，不是文字是否华丽。

### 3.4 图形与可渲染内容区

请做一个“可视化展示”章节，至少包含以下内容中的每一项：

- 脑图
- 流程图
- 时序图
- 甘特图
- 类关系图
- 用户游历图
- Git 历史图
- 实体关系图
- 图表
- 五线谱
- 一种 Graphviz 图
- 一种 PlantUML 图

要求这些内容在最终文档中应被视为可渲染的结构化内容，而不是普通代码样例堆砌。

### 3.5 富媒体与嵌入区

请增加一个“富内容展示”章节，至少包含：

- 一段 HTML 内容
- 一个外部嵌入区域
- 一个视频内容区
- 一个音频内容区

### 3.6 轻量结构化展示区

请增加一个“数据整理”章节，至少包含：

- 一个轻量的二维信息展示区，用于展示少量字段
- 一个真正用于记录结构化条目的区域，用于展示多条记录

这两者都要出现，但不要把它们做成同一种东西。

### 3.7 语义能力区

请增加一个“知识管理语义”章节，至少包含：

- 一个适合后续检索的标签使用示例
- 一个适合“收藏 / 稍后查看 / 收集已有内容”的示例
- 一个带有补充说明的块级信息样例
- 一组与复习相关的内容，其中至少体现 2 种不同的问答/记忆组织方式

注意：这里测试的是“语义是否放对位置”，不是只看视觉效果。

---

## 4. 验收重点

完成后，请自行检查结果是否满足以下要求：

1. 所有大类元素都已实际落地。
2. 没有把相近概念错误替代。
3. 没有把本该是结构化内容的部分退化成普通段落。
4. 没有把本该是排版结构的问题误当成样式问题。
5. 没有把本该是语义标记的问题误当成普通文本装饰。
6. 文档整体可读、分区清晰，不是零散元素罗列。

---

## 5. 判定标准

如果 AI 只是做出“看起来差不多”的效果，但元素类型、组织方式或语义归类不正确，则本轮测试应视为**不通过**。

如果 AI 能够根据 MCP 帮助资源，把这些元素放到正确的思源表达方式中，同时避免常见混淆，则可视为**通过**。
